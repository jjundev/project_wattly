# 15 — 접근성

> 막힘: 02,04 · 커버: 스토리 18 · 결정표 매핑: #13(상태), DS 접근성
> 프로토타입 근거: 전 카드 라벨/값/사유 구조 · DS 포커스 링 규칙([README](../interactive/project/_ds/wanted-design-system-019dded1-528c-7148-a04f-66cac240e314/README.md))

## 목표

VoiceOver 사용자가 메뉴바 라벨과 각 수치를 화면 낭독으로 들을 수 있다.

## 범위 (In)

1. **메뉴바** — 접근성 라벨(예 "Wattly, CPU 42 퍼센트"). 텍스트 OFF여도 라벨 제공.
2. **카드** — 각 카드에 이름·현재값·단위·(미가용 시)사유를 VoiceOver 값으로. 예 "프로세서 전력, 8.4 와트", "CPU 온도, 사용 불가, 검증된 센서 없음".
3. **부분 실패/콜드** — `loading`은 "불러오는 중", `unavailable`은 사유 문구(retryable은 "재시도 중", terminal은 사유). private key/kern code 노출 금지([08](08-temperature-cpu-gpu-battery.md)).
4. **스파크라인** — 장식으로 숨기거나 "최근 60초 추이" 요약만 제공(중복 낭독 방지).
5. **컨트롤** — 토글/세그먼트/슬라이더/✎/⚙︎에 역할·상태 라벨(켜짐/꺼짐, 현재 선택). DS **포커스 2px 링**(accent, 2px offset) 가시화.
6. **펼침** — CPU 코어·메모리 Top-3 행도 라벨(예 "S0 코어, 58 퍼센트").

## 범위 (Out)

- 음성 제어 커스텀 명령. 점자 디스플레이 전용 포맷.

## 수용 기준

- VoiceOver로 메뉴바→팝오버→각 카드→설정 컨트롤을 순회하며 값·상태를 모두 들을 수 있다.
- 미가용/콜드 상태가 사유와 함께 낭독된다.
- 키보드 포커스 링이 항상 보인다.

## 메모

- 접근성 라벨 문자열은 [14](14-menubar-text-metrics.md)의 조립 로직과 공유 가능(테스트 대상).

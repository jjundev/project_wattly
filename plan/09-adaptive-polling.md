# 09 — 적응형 폴링 + 저전력

> 막힘: 01,03 · 커버: 스토리 11,19 · 결정표 매핑: #17
> 프로토타입 근거: 업데이트 주기 세그먼트 line 316–323, 힌트 카피 line 322

## 목표

이 모니터링 앱 자체가 배터리를 거의 쓰지 않게 한다(전력 도구가 전력을 많이 쓰면 모순). 유휴 시 폴링을 낮추는 적응형 동작.

## 범위 (In)

1. **적응형 주기**
   - 패널 닫힘 = 5초(메뉴바 텍스트 OFF) / 2초(텍스트 ON).
   - 패널 열림 = 1–2초.
   - 설정 세그먼트로 상한 조절: **자동 / 1초 / 2초 / 5초**(`@AppStorage`). 자동이 위 적응형 동작.
2. **타이머/스케줄**
   - `Timer`에 `tolerance` 부여(코얼레싱). 폴링 Task QoS `.utility`.
3. **렌더/IO 생략**
   - 패널 닫힘 → 그래프 렌더 정지([03](03-sparkline-history.md)).
   - CPU/GPU 온도 둘 다 OFF → `TemperatureProvider` 호출·sensor I/O 생략([08](08-temperature-cpu-gpu-battery.md)).
   - 가시 토글이 꺼진 지표는 폴링에서 제외.
4. **설정 힌트 카피**(프로토타입 그대로) — "자동: 패널 열림 1–2초, 닫힘 5초(텍스트 ON 시 2초)로 낮춰 배터리를 아낍니다."

## 범위 (Out)

- 자체 전력 측정·회귀 가드([16](16-self-power-guard.md)). Metal 가속(저전력과 충돌, 명시 제외).

## 수용 기준

- 패널 열림/닫힘, 메뉴바 텍스트 ON/OFF에 따라 폴링 주기가 규칙대로 변한다.
- 온도 두 토글 OFF 시 sensor I/O가 실제로 발생하지 않는다(가짜 transport로 호출 카운트 검증, [18](18-testing.md)).
- 주기·패널 상태와 무관하게 60초 history가 유지된다([03](03-sparkline-history.md)).
